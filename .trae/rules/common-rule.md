## 1. 输出语言规则

- 所有对话回复、思考过程中的计划步骤、代码解释、注释内容，必须使用**简体中文**；
- 若涉及技术术语，需补充中文释义（如：“BEM 命名法（块-元素-修饰符，一种 CSS 命名规范）”）

## 2.样式规范

- 优先使用 Tailwind CSS 进行样式开发，确保样式一致性、可维护性与开发效率。避免冗余自定义样式，除非 Tailwind 无法满足特定设计需求。

## 3.兼容性要求

- 需严格处理 JavaScript 运行时边界值，对 null、undefined、NaN 及空字符串等空值场景进行显式校验与兜底，确保逻辑在任何数据状态下均不抛错、不白屏
- 所有代码必须兼容主流浏览器（Chrome、Edge、Safari、Firefox）及其近三个稳定版本。避免使用实验性或未广泛支持的语法与 API，确保生产环境稳定运行。
- 避免使用浏览器不支持的最新 CSS 特性，必要时需使用 polyfill 或 fallback 方案。

## 4. 文件与模块组织

- 每个功能模块或组件，应独立封装为一个单独的文件（如：`Button.tsx`、`Modal.js`），避免将所有代码写在一个文件中。
- 目录结构应清晰、合理，避免过深或过宽的层级嵌套，保持代码组织的可维护性。
